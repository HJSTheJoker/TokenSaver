# Contributing

## Development workflow

- Keep changes small and scoped.
- Prefer additive changes over risky rewrites.
- Preserve the backup-first installer behavior.
- Do not persist raw logs or prompt contents.

## Commands

```bash
swift build
swift test
```

## Style

- Swift 6 strict concurrency is enabled.
- Favor small typed models and explicit behavior over clever abstractions.
- Keep UI compact and native-first.
